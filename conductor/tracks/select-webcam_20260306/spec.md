# Specification: Select Webcam

**Track ID:** select-webcam_20260306
**Type:** Feature
**Created:** 2026-03-06
**Status:** Draft

## Summary

Allow users to choose which webcam is used for drinking detection when multiple cameras are connected.

## Context

Water Tracker uses AVFoundation to capture video frames for Vision-based drinking detection. Currently it always uses the system default video device. Users with multiple cameras (built-in + external) need to select which one to use.

## User Story

As a Mac user with multiple cameras, I want to choose which webcam is used for drinking detection so that tracking works reliably with my preferred camera.

## Acceptance Criteria

- [ ] A camera picker in Settings shows all available video capture devices by name
- [ ] The selected camera persists across app restarts (stored in AppSettings)
- [ ] WebcamMonitor uses the selected camera instead of the system default
- [ ] If the selected camera is disconnected, fall back to the default camera and show a notice
- [ ] Hot-plugging: if a new camera is connected/disconnected, the picker updates without restarting the app

## Dependencies

Builds on existing code: `WebcamMonitor`, `AppSettings`, `SettingsView`.

## Out of Scope

- Camera preview/thumbnail in the picker (calibration window already handles preview)
- Per-camera calibration profiles
- Audio device selection

## Technical Notes

- Use `AVCaptureDevice.DiscoverySession` to enumerate `.video` devices
- Store the selected device's `uniqueID` in `AppSettings`
- Observe `AVCaptureDevice` connect/disconnect notifications for hot-plug support
- Key change: `WebcamMonitor.configureSessionOnBackground()` currently hardcodes `AVCaptureDevice.default(for: .video)`
