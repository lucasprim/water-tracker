# Specification: UX Overhaul

**Track ID:** ux-overhaul_20260306
**Type:** Feature
**Created:** 2026-03-06
**Status:** Draft

## Summary

A comprehensive UX overhaul of the Water Tracker app, replacing the utilitarian interface with a polished, delightful experience that uses modern design patterns (circular progress ring, micro-interactions, sound feedback, celebrations) to keep users motivated and engaged with their hydration habit.

## Context

Water Tracker is a macOS menu bar app for logging daily water consumption. The current UI is functional but plain — a tapered cup visualization, a single "Log Bottle" button, settings embedded in the popover, no history/charts, and no satisfying feedback when logging drinks. Research into best-in-class menu bar apps (iStat Menus, Dato, Hand Mirror) and leading water tracking apps (WaterMinder, Waterllama) reveals significant opportunities to improve engagement through better visualization, micro-interactions, gamification (streaks), and data presentation.

## User Story

As a Water Tracker user, I want a polished, delightful interface with satisfying feedback and progress visualization so that I stay motivated to maintain my hydration habit.

## Acceptance Criteria

- [ ] Circular progress ring with wave fill animation replaces the cup visualization
- [ ] Multiple preset bottle sizes (e.g. 250ml, 350ml, 500ml, 750ml) as quick-log buttons on the main popover
- [ ] Sound + scale animation feedback when logging a drink
- [ ] Celebration animation (confetti/splash) when daily goal is reached
- [ ] Streak tracking displayed on the main view (flame icon + count)
- [ ] Weekly bar chart (Swift Charts) in an expandable section
- [ ] Settings moved to a proper Settings window (Cmd+,) with TabView
- [ ] Dynamic menu bar icon reflecting daily fill level
- [ ] Global keyboard shortcut for quick-logging default bottle size
- [ ] Undo toast shown after logging with visible "Undo" action
- [ ] Popover widened to ~320px with .thinMaterial background
- [ ] Detection log moved out of Settings into its own accessible section

## Dependencies

Depends on existing code:
- **WaterCupView** — will be replaced by the new circular progress ring
- **PopoverContentView** — will be heavily reworked (layout, width, buttons, undo toast)
- **SettingsView** — will be moved from popover-embedded to a standalone Settings window
- **WaterTrackerApp** (MenuBarExtra scene) — menu bar icon changes, Settings scene addition, keyboard shortcut
- **AppSettings / WaterEntry models** — will need new fields (preset sizes, streak data)
- **DailyProgressStore** — will need streak calculation logic
- **DrinkTimerManager / AppCoordinator** — integration with new feedback (sound, animation triggers)

## Out of Scope

- No new drink types (coffee, tea, etc.) — water only
- No Apple Watch companion app
- No CloudKit sync
- No onboarding/first-run wizard
- No changes to webcam detection algorithms or calibration workflow
- No changes to the notification/reminder system (screen-edge overlay stays as-is)
- No monthly/long-term views (heat map, trend lines) — just weekly chart for now

## Technical Notes

- **Swift Charts** for the weekly bar chart (already available on macOS 14+)
- **SwiftUI `Settings` scene** for the standalone settings window
- **`NSSound`** for logging feedback sounds (respect system sound preference)
- **Canvas + TimelineView** for the circular progress ring with wave fill (similar pattern to existing WaterCupView)
- **`@AppStorage`** for new user preferences (preset sizes, sound enabled, keyboard shortcut)
- **SF Symbols** for dynamic menu bar icon (`drop.fill` variants or custom rendering)
- **`NSEvent.addGlobalMonitorForEvents`** or `KeyboardShortcuts` SPM package for the global hotkey
- **No new dependencies** beyond what's already in the project, except possibly a small SPM package for confetti animation (or hand-roll it with CAEmitterLayer)
- **Incremental migration**: Keep existing models backward-compatible, add new fields with defaults so existing SwiftData entries aren't lost
